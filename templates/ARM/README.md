# VM Fleet Commander Project
## ARM Template

This folder will contain version control for all ARM templates.


### Tasks include:

1. Initial Template:

Set up a version control system (e.g., Git) to track changes in my ARM templates.

2. ARM Basics:

Start with learning the basics of ARM syntax and structure.

3. Resource Group and Naming Conventions:

Define a ARM file to create an Azure Resource Group for your VMs.
Implement naming conventions for your resources using ARM's string functions.

4. Virtual Machine Provisioning:

Create a ARM module for deploying Azure VMs.

5. Network Resources:

Design ARM modules for associated networking resources like Virtual Network, Subnet, and Network Security Groups.
Ensure your VMs are provisioned within the designated VNet and have the necessary security rules applied.

6. Parameter Files and Validation:

Create separate parameter files for your ARM templates, allowing for different environment deployments (e.g., dev, test, prod).

7. Deployment:

Use the Azure CLI to deploy your ARM templates, creating all designated resources.

8. Maintenance & Updates:

Make changes to your ARM files (e.g., VM size or count) and redeploy.

9. Documentation & Cleanup:

Document your ARM modules, their purpose, and any parameters required.
